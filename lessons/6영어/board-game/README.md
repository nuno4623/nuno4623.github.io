# 6학년 영어 부루마블 — 교사용 안내

같은 폴더의 `index.html` 하나로 동작하는 게임입니다. 파일을 열면 세 가지 모드를 고를 수 있습니다.

- **개인전**: 학생 1명 + 봇 3명. Firebase 설정 없이 항상 바로 동작합니다.
- **모둠전**: 태블릿 1대를 모둠이 돌려가며 하는 핫시트 모드. Firebase 설정 없이 항상 바로 동작합니다.
- **같이하기(멀티)**: 학생마다 자기 기기로 접속해 실시간으로 함께 플레이. **Firebase 설정을 마쳐야 탭이 나타납니다.**

즉, Firebase를 설정하지 않아도 개인전·모둠전은 지금 그대로 배포해서 쓸 수 있습니다. "같이하기"만 아래 설정이 필요합니다.

---

## 1. 같이하기(멀티) 모드 켜기 — Firebase 설정 (교사용, 1회)

### 1) 프로젝트 만들기
1. [Firebase 콘솔](https://console.firebase.google.com) 접속 → Google 계정으로 로그인.
2. **프로젝트 추가** 클릭.
3. 프로젝트 이름 입력(예: `우리반-영어부루마블`) → 계속.
4. Google Analytics는 **사용 안 함**으로 두고 → 프로젝트 만들기.

### 2) Realtime Database 만들기
1. 왼쪽 메뉴에서 **빌드 → Realtime Database** 클릭.
2. **데이터베이스 만들기** 클릭.
3. 위치는 기본값(또는 가까운 지역) 선택 → 다음.
4. 보안 규칙은 일단 **테스트 모드로 시작** 선택 → 사용 설정.
   (규칙은 아래 3번 단계에서 다시 교체할 것입니다.)

### 3) 보안 규칙 교체 (중요)
1. Realtime Database 화면에서 위쪽 탭 중 **규칙** 클릭.
2. 아래 내용으로 전체 교체 후 **게시** 클릭:

```json
{
  "rules": {
    "rooms": {
      "$roomCode": {
        ".read": true,
        ".write": true
      }
    },
    "$other": {
      ".read": false,
      ".write": false
    }
  }
}
```

이 규칙은 `rooms/방코드` 경로만 누구나 읽고 쓸 수 있게 하고, 그 외 모든 경로는 차단합니다. 방 코드를 모르면 다른 반 데이터에 접근할 수 없습니다.

### 4) 웹 앱 등록 후 설정값(config) 6개 복사
1. 왼쪽 위 **⚙️ 프로젝트 설정** 클릭 → **일반** 탭.
2. 아래로 스크롤 → **내 앱** 항목에서 **</> (웹)** 아이콘 클릭.
3. 앱 닉네임 입력(예: `board-game`) → Firebase Hosting은 체크하지 않음 → **앱 등록**.
4. 화면에 아래와 비슷한 코드가 나타납니다. 이 안의 값들을 그대로 복사해 둡니다.

```js
const firebaseConfig = {
  apiKey: "AIza...",
  authDomain: "우리반-영어부루마블.firebaseapp.com",
  databaseURL: "https://우리반-영어부루마블-default-rtdb.firebaseio.com",
  projectId: "우리반-영어부루마블",
  storageBucket: "우리반-영어부루마블.appspot.com",
  messagingSenderId: "1234567890",
  appId: "1:1234567890:web:abcdef123456"
};
```

### 5) index.html에 붙여넣기
1. 이 폴더의 `index.html` 파일을 텍스트 에디터로 엽니다.
2. `FIREBASE_CONFIG` 를 찾습니다 (스크립트 상단, `SUBMIT_FORM` 바로 아래에 있습니다).
3. 7개 값(`apiKey`, `authDomain`, `databaseURL`, `projectId`, `storageBucket`, `messagingSenderId`, `appId`)을 4번에서 복사한 값으로 채웁니다.

```js
const FIREBASE_CONFIG = {
  apiKey: "AIza...",
  authDomain: "우리반-영어부루마블.firebaseapp.com",
  databaseURL: "https://우리반-영어부루마블-default-rtdb.firebaseio.com",
  projectId: "우리반-영어부루마블",
  storageBucket: "우리반-영어부루마블.appspot.com",
  messagingSenderId: "1234567890",
  appId: "1:1234567890:web:abcdef123456"
};
```

4. 저장 후 커밋·푸시하면 1~2분 뒤 배포 사이트에 반영됩니다. 값이 모두 채워지면 시작 화면에 **"🌐 같이하기"** 탭이 자동으로 나타납니다. (값이 비어 있으면 탭이 숨겨지고 개인전·모둠전만 보입니다.)

### 같이하기는 어떻게 동작하나요?
- 한 명이 **방 만들기**를 누르면 4자리 방 코드가 생기고, 나머지 학생들이 **방 코드로 입장**해서 들어옵니다.
- 2~4명이 모이면 방장(방을 만든 학생) 화면에만 **"게임 시작!"** 버튼이 나타납니다.
- 자기 턴이 아닐 때는 주사위 버튼이 잠기고, 상대 턴이 끝나면 자동으로 점수·위치가 화면에 반영됩니다.
- 새로고침해도 같은 방에 자동으로 다시 연결됩니다.
- 한 학생이 45초 넘게 반응이 없으면 방장 기기가 자동으로 다음 팀으로 턴을 넘깁니다.
- 게임이 끝난 방 데이터는 자동으로 지워지지 않습니다(Realtime Database에는 자동 만료 기능이 없습니다). 같은 방 코드로 새로 방을 만들면 기존 데이터를 덮어쓰므로 평소엔 신경 쓰지 않아도 되고, 데이터가 많이 쌓였다 싶으면 Firebase 콘솔의 Realtime Database 화면에서 `rooms` 아래 오래된 항목을 수동으로 삭제하면 됩니다.

## 2. 점수 자동 집계 — Google Form 설정 (선택)

개인전·같이하기 모드는 게임이 끝나면 결과 화면에 **"📨 점수 제출"** 버튼이 나타날 수 있습니다(모둠전에는 없습니다 — 한 기기를 여럿이 같이 쓰기 때문입니다). 이 버튼으로 학생이 자기 반/번호/점수를 구글 폼에 자동 제출하게 하려면 아래를 설정하세요. 설정하지 않으면 버튼이 그냥 나타나지 않을 뿐, 게임 자체는 정상 동작합니다.

### 1) Google Form 만들기
1. [Google Forms](https://forms.google.com)에서 새 설문지 만들기.
2. 아래 5개 항목을 **단답형**으로 추가 (순서·제목은 자유):
   - 반
   - 번호
   - 팀
   - 점수
   - 획득한 땅 수

### 2) URL과 항목(entry) ID 확인하기
1. 만든 폼을 미리보기(오른쪽 위 눈 모양 아이콘)로 엽니다.
2. 각 항목에 아무 값이나 입력하고 **제출**합니다.
3. 제출 후 주소창의 URL을 복사합니다 — `.../viewform` 을 `.../formResponse` 로 바꾼 것이 제출용 URL입니다.
4. 각 항목의 entry ID를 확인하려면: 미리보기 페이지에서 마우스 우클릭 → **페이지 소스 보기** → `entry.` 로 검색하면 `entry.123456789` 같은 숫자가 각 항목마다 나옵니다. 어떤 항목의 ID인지는 근처의 질문 텍스트로 구분합니다.
   (더 쉬운 방법: 브라우저 개발자 도구 없이도, 폼 편집 화면에서 각 질문을 클릭했을 때 우측 패널에 나오는 정보로는 확인이 어려우니 위 소스보기 방법을 권장합니다.)

### 3) index.html에 붙여넣기
1. `index.html`에서 `SUBMIT_FORM` 을 찾습니다 (스크립트 맨 위).
2. `url`에 2-3에서 만든 `.../formResponse` 주소를, `entries`의 각 항목에 해당 entry ID를 채웁니다.

```js
const SUBMIT_FORM = {
  url: "https://docs.google.com/forms/d/e/여기에_폼_ID/formResponse",
  entries: {
    cls:   "entry.111111111",   // 반
    num:   "entry.222222222",   // 번호
    team:  "entry.333333333",   // 팀
    score: "entry.444444444",   // 점수
    lands: "entry.555555555"    // 획득한 땅 수
  }
};
```

3. 저장 후 커밋·푸시. 값이 모두 채워지면 결과 화면에 제출 버튼이 자동으로 나타납니다.
4. 제출된 값은 폼과 연결된 스프레드시트(응답 탭 → 스프레드시트 만들기)에서 확인할 수 있습니다.
